## What went well

- **Spec-then-verify on the layout trap.** create-story flagged the AD-19 root-scoping risk before coding; the dev verified Hugo's lookup behavior empirically (a marker test) rather than trusting the spec, and found the override must live at the docs `list.html` slot (type beats section under `cascade type:docs`). Fresh-eyes code review: Approve-with-nits.
- **UAT caught what automation didn't.** Carrie's deliberate human UAT step found the mobile sidebar break, the empty section pages, and the footer scrollbar — none of which a build check would catch. This validated the story-by-story + UAT design for Epic 3 (vs. the autonomous-to-PR style of Epics 1–2).
- **Zero-upkeep section cards.** The "In this section" cards auto-derive from `.Pages` by weight — new topics and stub-flips reflect automatically.
- **Clean go-live.** A single `main` merge; production verified end-to-end (HTTP 200, sitemap, GoatCounter cookieless/prod-only, no consent banner).
- **Voice iteration landed.** Welcome copy was reworked across several passes to match Carrie's real blog voice (and saved to memory so future copy matches).

## Challenges & lessons

1. **Code review didn't test viewport sizes.** *(Carrie's key observation.)* The 3.1 review was strong on logic/ACs/a11y/the root-guard but never exercised mobile/tablet/desktop widths. A latent Epic-1 bug (sidebar gone on mobile, no drawer tree) only surfaced in Carrie's UAT. **Lesson:** make responsive behavior — especially mobile — an explicit AC and a required code-review dimension, not an assumption. (Saved as a working preference; see action item 4.)
2. **A spec path was mechanically impossible.** epics/spine said `layouts/handbook/list.html`; under `cascade type:docs` that template never runs. **Lesson:** verify framework lookup behavior empirically; spec the slot, not a guess. (Corrected across epics, spine, cheatsheet, readiness report.)
3. **Frozen theme copy = drift risk.** The section-list else-branch is a verbatim copy of Hextra's `docs/list.html`. **Lesson:** copying theme internals is a maintenance liability; pinned version noted in-file, but a Hextra bump needs a re-diff.
4. **Decisions lived only in chat.** "The paragraphs we talked about" from a prior session were never persisted, so the welcome copy had to be reconstructed. **Lesson:** persist agreed copy/decisions into artifacts. (Now saving copy into the story files.)
5. **Led with my own diagnosis over the user's hypothesis.** On the scrollbar, I proposed a cause before explicitly testing Carrie's stated hypothesis (the hidden toggle); she pushed back. **Lesson:** test the user's hypothesis first, report, then propose alternatives. (Saved as a working preference.)
